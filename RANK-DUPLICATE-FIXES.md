# 排行榜重复用户问题修复说明

## 问题描述

用户反馈排行榜出现以下问题：
1. **重复的用户**：例如第一名也会出现在下方列表中
2. **排名排序不正确**：排名没有按正确顺序显示

## 问题分析

### 1. 重复用户问题原因
- **WX:KEY绑定错误**：WXML中使用 `wx:key="id"`，但云函数返回的是 `_id` 字段
- **当前用户重复显示**：当前用户既在排行榜列表中，又作为"当前用户"单独显示在下方

### 2. 排名排序问题原因
- **相同carbon值**：多个用户可能有相同的 `totalCarbon` 值
- **排序不稳定**：数据库对相同值的排序可能随机
- **排名计算**：分页排名计算可能导致不连续

## 修复内容

### 1. WXML修复 (pages/rank/rank.wxml)

#### 1.1 修复WX:KEY绑定
```xml
<!-- 修改前 -->
wx:key="id"

<!-- 修改后 -->
wx:key="_id"
```

#### 1.2 修复用户ID绑定
```xml
<!-- 修改前 -->
data-user-id="{{item.id}}"
data-user-id="{{rankList[2] && rankList[2].id}}"

<!-- 修改后 -->
data-user-id="{{item._id}}"
data-user-id="{{rankList[2] && rankList[2]._id}}"
```

### 2. JS修复 (pages/rank/rank.js)

#### 2.1 避免当前用户重复显示
```javascript
// 检查当前用户是否已经在排行榜列表中
let finalRankList = formattedRankList
let finalUserRank = formattedUserRank

// 如果当前用户有排名，并且已经在排行榜列表中，从列表中移除，避免重复显示
if (formattedUserRank && formattedUserRank.userId) {
  // 查找当前用户是否在排行榜列表中
  const userIndex = formattedRankList.findIndex(item => item._id === formattedUserRank.userId)
  if (userIndex !== -1) {
    // 从列表中移除当前用户，因为会在下方单独显示
    finalRankList = formattedRankList.filter(item => item._id !== formattedUserRank.userId)
    // 确保排名顺序正确，重新计算排名
    finalRankList.forEach((item, index) => {
      // 保持原有相对排名，但确保排名数字连续
      const baseRank = (page - 1) * limit + 1
      item.rank = baseRank + index
    })
  }
}
```

### 3. 云函数修复 (cloudfunctions/get-rank/index.js)

#### 3.1 添加二级排序确保稳定
```javascript
.orderBy('totalCarbon', 'desc')
.orderBy('_id', 'asc') // 添加二级排序，确保相同carbon值时的稳定排序
```

## 修复效果

### 修复前的问题
1. **重复显示**：当前用户可能同时出现在排行榜列表和"当前用户"区域
2. **渲染问题**：WX:KEY绑定错误导致列表渲染异常
3. **排序不稳定**：相同carbon值的用户可能随机排序

### 修复后的效果
1. **唯一显示**：当前用户只显示在"当前用户"区域，不在列表中重复
2. **正确渲染**：使用正确的 `_id` 字段作为WX:KEY
3. **稳定排序**：相同carbon值时按 `_id` 升序排序，确保每次查询结果一致
4. **排名连续**：移除当前用户后重新计算排名，确保排名数字连续

## 技术细节

### 1. WX:KEY的重要性
- `wx:key` 帮助小程序识别列表项的唯一性
- 错误的key会导致列表渲染异常、性能下降
- 使用数据库主键 `_id` 是最佳实践

### 2. 当前用户处理逻辑
- **检查存在**：查找当前用户是否在排行榜列表中
- **移除重复**：从列表中移除当前用户
- **重新排名**：确保排名数字连续
- **单独显示**：在"当前用户"区域突出显示

### 3. 数据库排序优化
- **主排序**：`totalCarbon` 降序（碳减排量从高到低）
- **二级排序**：`_id` 升序（相同carbon值时按ID排序）
- **稳定性**：确保每次查询结果相同
